# 个人主页开发与部署指南

## 一、技术选型与依据

### 1. 前端：React + Vite

| 选择             | 依据                                                                                  |
| ---------------- | ------------------------------------------------------------------------------------- |
| **React**        | 组件化、生态成熟、求职市场主流，便于维护和扩展。                                      |
| **Vite**         | 官方推荐构建工具，开发时 ES 模块原生加载、HMR 极快，生产构建基于 Rollup，打包体积小。 |
| **React Router** | 单页应用路由标准方案。                                                                |
| **现代 CSS**     | 使用 CSS Modules 或 Tailwind，避免全局污染，便于响应式。                              |

### 2. 后端：Node.js + Express

| 选择         | 依据                                                                       |
| ------------ | -------------------------------------------------------------------------- |
| **Node.js**  | 与前端同语言，前后端可共享类型/工具；部署简单，适合个人项目。              |
| **Express**  | 最常用的 Node 框架，文档多、中间件丰富，适合 API、联系表单、简单业务逻辑。 |
| **REST API** | 前后端分离，前端通过 `fetch`/axios 调用；便于以后加移动端或第三方集成。    |

### 3. 项目结构建议

```
feng's webpage/
├── frontend/          # React 前端 (Vite)
├── backend/           # Node.js 后端 (Express)
├── 个人主页开发与部署指南.md  # 本文档
└── README.md          # 项目总览与本地/部署说明
```

前后端分离便于：分别部署、独立扩展、前端可部署到 CDN/静态托管。

**本地端口、前后端怎么连、澳大利亚服务器怎么租、代码逐段解释** → 见 **[开发说明-本地与连接.md](./开发说明-本地与连接.md)**。

## 二、分步实施计划

### 阶段一：本地开发环境

1. **安装 Node.js**  
   从 [nodejs.org](https://nodejs.org/) 安装 LTS 版本（建议 18+）。  
   验证：`node -v`、`npm -v`。

2. **创建前端项目（React + Vite）**
   - 在项目根目录下执行：  
     `npm create vite@latest frontend -- --template react`
   - 进入 `frontend`，执行：`npm install`。
   - 可选：安装路由与样式库，例如：  
     `npm install react-router-dom`，或按需安装 Tailwind。

3. **创建后端项目（Express）**
   - 在项目根目录下创建 `backend` 文件夹。
   - 初始化：`npm init -y`。
   - 安装依赖：`npm install express cors dotenv`。
   - 编写入口文件（如 `server.js`），提供健康检查接口（如 `GET /api/health`）和联系表单接口（如 `POST /api/contact`）。

4. **本地联调**
   - 前端：`cd frontend && npm run dev`（默认如 http://localhost:5173）。
   - 后端：`cd backend && node server.js`（如 http://localhost:3000）。
   - 前端请求后端时使用 `http://localhost:3000` 作为 API 基础地址；后端需开启 CORS 允许前端源。

### 阶段二：功能与页面

5. **前端页面结构建议**
   - 首页：Hero（姓名、一句话介绍）+ 导航。
   - 关于：简介、技能、教育/经历摘要。
   - 项目/作品：列表或卡片，可链接到 GitHub/在线 Demo。
   - 简历：可嵌入 PDF 或链接到你的 Wayne-Lin-Resume.pdf。
   - 联系：表单（姓名、邮箱、留言），提交到后端 `/api/contact`。

6. **后端接口**
   - `GET /api/health`：健康检查，部署后用于探活。
   - `POST /api/contact`：接收表单，校验后可选：写文件、发邮件或接入第三方（如 SendGrid）。
   - 若暂无邮件服务，可先返回成功并写入本地 JSON/文本日志，便于后续接入。

### 阶段三：部署

7. **部署方式选型（简要）**

   | 方案                | 前端                                        | 后端                              | 特点                           |
   | ------------------- | ------------------------------------------- | --------------------------------- | ------------------------------ |
   | A. Vercel + Railway | Vercel 部署 React                           | Railway 部署 Node                 | 免费额度够用，配置简单         |
   | B. 单机/云服务器    | Nginx 托管 build 或 Node 静态               | PM2 跑 Express                    | 完全自控，需自己装 Node、Nginx |
   | C. 全 Vercel        | Vercel 部署 React，API 用 Vercel Serverless | 无独立后端或 Serverless Functions | 最简单，适合纯静态+轻量 API    |

8. **推荐起步：Vercel（前端）+ Railway（后端）**
   - 前端：在 Vercel 连接 Git 仓库，根目录选 `frontend`，构建命令 `npm run build`，输出目录 `dist`。
   - 后端：在 Railway 连接同一仓库，根目录选 `backend`，启动命令 `node server.js`。
   - 环境变量：前端写 `VITE_API_URL=https://你的后端地址`；后端如需发邮件则配置 `SMTP_*` 等。

9. **上线后检查**
   - 浏览器打开前端 URL，检查首页、关于、项目、简历、联系页。
   - 提交联系表单，确认后端收到请求（日志或存储）。
   - 检查控制台无 CORS 错误、网络无 404。

## 三、安全与性能建议

- **后端**：对 `POST /api/contact` 做速率限制（如 express-rate-limit），防止滥发；校验邮箱格式与必填项。
- **前端**：API 地址通过环境变量注入，不要写死生产域名。
- **HTTPS**：Vercel、Railway 默认 HTTPS，若自建服务器请用 Nginx 配 SSL（如 Let’s Encrypt）。

## 四、下一步

按下面顺序进行即可：

1. 在本地创建并运行 **frontend**（React + Vite）。
2. 在本地创建并运行 **backend**（Express），并实现健康检查与联系接口。
3. 前端调用后端 API，完成联系表单与基础页面。
4. 按本文「阶段三」选择一种方式部署并测试。

本仓库中已为你准备好 **frontend** 与 **backend** 的初始代码骨架，可直接在本地运行并按需修改内容与样式。
