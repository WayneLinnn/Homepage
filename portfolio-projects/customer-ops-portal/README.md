# Customer Ops Portal

Full-stack business system blueprint for backend or backend-leaning full-stack applications.

## Purpose

Show that I can build the sort of workflow software many Australian SMEs actually buy:

- customer intake
- internal task tracking
- status updates
- audit-friendly notes
- operational reporting

## Recommended stack

- Backend: Java, Spring Boot, JPA, PostgreSQL
- Frontend: React, TypeScript
- Delivery: Docker, AWS-ready deployment

## MVP scope

- Login and role-based access
- Customer records
- Task workflow with status history
- Notes and follow-ups
- Simple dashboard metrics

## Hiring signal

This project helps me explain:

- API design
- business entity modelling
- auth and permissions
- frontend/backend integration
- how I scope an MVP for a real customer problem
