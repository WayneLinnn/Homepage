# 如何查看你的个人主页（详细流程）

## 一、为什么打开 index.html 看不到内容？

在 React + Vite 项目里，**index.html 故意做得很简单**，只做两件事：

1. 放一个空的 `<div id="root"></div>`（占位）
2. 用 `<script src="/src/main.jsx"></script>` 加载入口脚本

**真正的页面内容**（Hero、About、Experience、Projects、Contact 等）**不是写在 HTML 里的**，而是由 **React** 在浏览器里根据 **main.jsx → App.jsx → Home / Layout 等组件** 动态生成，并“画”进 `#root` 里。

所以：

- **不能** 直接双击 `index.html` 用浏览器打开：那样没有开发服务器，浏览器无法正确加载 `/src/main.jsx` 这种模块路径，React 也不会运行，你只会看到一个空白页或报错。
- **必须** 在项目里先**安装依赖**，再**启动前端的开发服务器**，然后用浏览器访问 **http://localhost:5173**，才能看到完整主页。

---

## 二、正确查看主页的步骤（按顺序做）

### 第 1 步：打开终端

- **Windows**：在 VS Code / Cursor 里按 `` Ctrl+` `` 打开集成终端；或在“开始”里搜“终端”或“PowerShell”打开。
- 确保当前目录是项目根目录（例如 `c:\Users\LF\Desktop\feng's webpage`）。  
  若不是，先执行：  
  `cd "c:\Users\LF\Desktop\feng's webpage"`

### 第 2 步：进入前端目录

在终端里执行：

```bash
cd frontend
```

执行完后，终端当前目录应为：`...\feng's webpage\frontend`。

### 第 3 步：安装依赖（首次必须，以后不用每次做）

在 **frontend** 目录下执行：

```bash
npm install
```

等待安装完成（可能一两分钟）。若报错，检查本机是否已安装 Node.js（在终端执行 `node -v` 应能看到版本号）。

### 第 4 步：启动前端开发服务器

仍在 **frontend** 目录下执行：

```bash
npm run dev
```

终端里会出现类似：

```text
  VITE v5.x.x  ready in xxx ms
  ➜  Local:   http://localhost:5173/
  ➜  press h + enter to show help
```

说明开发服务器已启动，在 **5173 端口** 提供页面。

### 第 5 步：用浏览器打开主页

1. 打开浏览器（Chrome、Edge、Firefox 等）。
2. 在地址栏输入：**http://localhost:5173**
3. 回车。

此时应能看到你的个人主页：Hero、About、Experience、Projects、Contact 等，以及顶部导航、页脚。

---

## 三、流程小结

| 步骤 | 在哪个目录 | 执行什么                       | 说明                       |
| ---- | ---------- | ------------------------------ | -------------------------- |
| 1    | 项目根目录 | 打开终端                       | 确保在 `feng's webpage` 下 |
| 2    | 项目根目录 | `cd frontend`                  | 进入前端项目               |
| 3    | frontend   | `npm install`                  | 首次必须，安装依赖         |
| 4    | frontend   | `npm run dev`                  | 启动开发服务器             |
| 5    | 浏览器     | 打开 **http://localhost:5173** | 查看主页                   |

**记住**：

- 看主页 = 先 `cd frontend` → `npm run dev` → 浏览器打开 **http://localhost:5173**。
- 不要用“双击 index.html”的方式查看，那样看不到 React 渲染的内容。

---

## 四、若想看“联系表单提交”等需要后端的部分

联系表单会请求后端 API。若你也要在本地测试提交：

1. **再开一个终端**（不要关掉正在跑 `npm run dev` 的那个）。
2. 在项目根目录执行：`cd backend`，然后 `npm install`（首次），再 `npm start`。
3. 后端会在 **http://localhost:3000** 运行；前端的联系表单会通过 Vite 代理把请求转到 3000，无需改地址。

这样前端（5173）+ 后端（3000）同时运行，即可在本地完整体验主页和表单。

---

## 五、index.html 里到底有什么（对照看）

你打开的 **frontend/index.html** 里只有：

- 一个空的 `<div id="root"></div>`：React 会把整站内容渲染进这里。
- 一行 `<script type="module" src="/src/main.jsx"></script>`：浏览器通过**开发服务器**访问时，Vite 会把 `main.jsx` 以及它引用的所有组件（App、Layout、Home、SectionHero、About、Experience、Projects、Contact 等）打包并执行，最后把界面“画”到 `#root` 里。

所以 **index.html 里“没有”Hero、About 等文字是正常的**——它们都在 **src/** 下的 .jsx 和 .css 文件里，由 React 在运行时生成。  
要看到这些内容，就必须按上面流程：**在 frontend 里执行 `npm run dev`，然后用浏览器打开 http://localhost:5173**。
